This directory contains the Python scripts used to compute and visualize the radial distribution functions (RDFs) and structure factors S(q) for the RuP MLMD simulations. The analysis is organized into four subfolders, covering both the MD trajectory data and the time-averaged structures.

---

## Overall Data Layout

All simulation data lives under a common root directory. The scripts in this folder read from and write to the temperature subdirectories following this layout:

```
<root>/
├── 10K/
│   └── lammps_out/
│       ├── rup_traj_sampled-50_100ps.xyz
│       ├── lattice_params.dat
│       ├── rup_traj_sampled-50_100ps_average_structure_10K.xyz
│       ├── rdf_analysis/          # partial RDF data computed by OVITO (MD trajectory)
│       └── avg_rdf_analysis/      # partial RDF data computed by OVITO (time-averaged structure)
├── 50K/
│   └── lammps_out/
│       └── ...
...
├── 700K/
│   └── lammps_out/
│       └── ...
└── data_analysis_dense_sampling/  # all analysis scripts live here
    ├── rdf_analysis/
    │   └── code_plot_rdf_ru-ru_opt_with_inset_2d-map.py
    ├── avg_rdf_analysis/
    │   └── code_plot_avg_rdf_ru-ru_opt_with_inset_2d-map.py

```

The scripts use relative paths of the form `../../{T}K/lammps_out/...` to reach the simulation data from within their respective subdirectories under `data_analysis_dense_sampling/`.

## 1. RDF — MD Trajectory (`./rdf_analysis/`)

Partial RDF data for each temperature is first computed from the MLMD trajectories using **OVITO** software and saved into each temperature's `lammps_out/rdf_analysis/` subdirectory.

Run from `<root>/data_analysis_dense_sampling/rdf_analysis/`:

```bash
python3 code_plot_rdf_ru-ru_opt_with_inset_2d-map.py
```

This script reads the pre-computed RDF data for all 23 temperatures, applies Gaussian smoothing, and produces a **2D temperature–distance heatmap** of the Ru–Ru RDF across the full temperature range, saved as `rdf_2d_heatmap.png`.

Intermediate RDF data is cached to disk (`.npz` format) on first run to accelerate subsequent re-plotting with different visual settings. To force recomputation, set `force_recompute=True` in the script.

### Key Script Parameters

| Parameter | Default | Description |
|-----------|---------|-------------|
| `distance_range` | (2.65, 3.75) Å | Ru–Ru distance window for the heatmap |
| `sigma` | 6.5 | Gaussian smoothing strength |
| `colormap` | `RdBu_r` | Matplotlib colormap for the heatmap |
| `temp_resolution` | 500 | Interpolated temperature grid points |
| `dist_resolution` | 500 | Interpolated distance grid points |
| `interpolation` | `linear` | Grid interpolation method |
| `horizontal_line_temps` | [320, 360] | Temperatures (K) where dashed marker lines are drawn |

### Input / Output

| | Path |
|---|---|
| **Input** (per temperature) | `../../{T}K/lammps_out/rdf_analysis/partial_rdfs.txt` |
| **Cache** | `./cache_rdf_2d_*.npz` |
| **Output** | `./rdf_2d_heatmap.png` |

## 3. Structure Factor — MD Trajectory (`./structure_factor/`)

### Step 1 — Compute S(q)

Run from `<root>/data_analysis_dense_sampling/structure_factor/`:

```bash
python3 code_sq_compute.py \
    -parent ../../ \
    -i rup_traj_sampled-50_100ps \
    -o ./sq_data/
```

This script reads the XYZ trajectories for all 23 temperatures, computes the instantaneous structure factor S<sub>insta</sub>(q) frame by frame along a high-symmetry q-path using the `dynasor` library, and saves one compressed `.npz` file per temperature to `./sq_data/`.

The q-path is defined in fractional coordinates of the monoclinic reference cell:

```
[100] → [101] → [111] → [011] → [001]
```

A relaxed monoclinic reference structure (`monoclinic_fully_relaxed.vasp`) must be present in the working directory to convert the fractional q-path to Cartesian coordinates.

Key command-line arguments:

| Argument | Default | Description |
|----------|---------|-------------|
| `-parent` | — | Path to the root directory containing temperature subfolders |
| `-i` | — | Pattern to match XYZ files (e.g., `rup_traj_sampled-50_100ps`) |
| `-o` | — | Output directory for `.npz` files |
| `-prefix` | `sq` | Output file prefix (produces `sq_10K.npz`, `sq_50K.npz`, …) |
| `-subdir` | `lammps_out` | Subdirectory within each temperature folder |
| `-ref` | `monoclinic_fully_relaxed.vasp` | Reference structure for q-path conversion |
| `-start` | 0 | First frame to process |
| `-step` | 1 | Frame stride |
| `-max` | None | Maximum number of frames per temperature |
| `-nq` | 50 | Number of q-points per segment |
| `-temps` | all 23 | Subset of temperatures to process |

Individual temperatures can be added without recomputing all results by passing e.g. `-temps 450K`.

### Step 2 — Plot S(q)

Run from `<root>/data_analysis_dense_sampling/structure_factor/`:

```bash
python3 code_sq_plot.py \
    -i ./sq_data/ \
    -o sq_plot.pdf
```

This script loads all `sq_*.npz` files, applies optional Gaussian smoothing, and plots ⟨S<sub>insta</sub>(q)⟩ along the high-symmetry q-path for all temperatures, colour-coded by temperature using the `coolwarm` colormap. Error shading (95% confidence interval) is shown by default.

Key command-line arguments:

| Argument | Default | Description |
|----------|---------|-------------|
| `-i` | — | Input directory containing `.npz` files |
| `-o` | — | Output figure file (`.pdf` or `.png`) |
| `-prefix` | `sq` | File prefix to load (`sq_*.npz`) |
| `-sigma` | 3.0 | Gaussian smoothing sigma (0 = no smoothing) |
| `-no-error` | off | Disable 95% CI shading |
| `-temps` | all | Select specific temperatures to plot |
| `-linear` | off | Use linear instead of log y-scale |
| `-cmap` | `coolwarm` | Matplotlib colormap |
| `-ylim` | None | Manual y-axis limits |

### Input / Output

| | Path |
|---|---|
| **Input** (per temperature) | `../../{T}K/lammps_out/rup_traj_sampled-50_100ps.xyz` |
| **Intermediate** | `./sq_data/sq_{T}K.npz` |
| **Output** | `./sq_plot.pdf` |

## 4. Structure Factor — Time-Averaged Structures (`./avg_structure_factor/`)

### Step 1 — Compute S(q)

Run from `<root>/data_analysis_dense_sampling/avg_structure_factor/`:

```bash
python3 code_sq_compute_avg.py \
    -parent ../../ \
    -i rup_traj_sampled-50_100ps_average_structure \
    -o ./sq_data_avg/
```

Operates identically to `code_sq_compute.py` but reads the time-averaged structure XYZ files (one static structure per temperature) and computes S<sub>avg</sub>(q) along the same high-symmetry q-path.

### Step 2 — Plot S(q)

Run from `<root>/data_analysis_dense_sampling/avg_structure_factor/`:

```bash
python3 code_sq_plot_avg.py \
    -i ./sq_data_avg/ \
    -o sq_avg_plot.pdf
```

Identical interface to `code_sq_plot.py`, but plots S<sub>avg</sub>(q) derived from the time-averaged structures.

### Input / Output

| | Path |
|---|---|
| **Input** (per temperature) | `../../{T}K/lammps_out/rup_traj_sampled-50_100ps_average_structure_{T}K.xyz` |
| **Intermediate** | `./sq_data_avg/sq_{T}K.npz` |
| **Output** | `./sq_avg_plot.pdf` |

## Requirements

- **Python**: ≥ 3.8
- **NumPy**: Array operations
- **Matplotlib**: Plotting and figure export
- **SciPy**: Gaussian smoothing (`scipy.ndimage`), spline interpolation (`scipy.interpolate`)
- **ASE**: Reading XYZ and VASP structure files
- **dynasor**: Reciprocal-space density kernel (`dynasor.core.reciprocal.calc_rho_q`)
- **tqdm**: Progress bar display
- **seaborn**: Imported by the RDF script
- **OVITO**: External software used to compute partial RDFs from LAMMPS XYZ files prior to running the Python scripts (see [ovito.org](https://www.ovito.org))
